# HOW TO BUILD THIS — Complete Step by Step Guide
# Read this entirely before you start.

==============================================================
WHAT YOU ARE BUILDING
==============================================================

Fasih-Docs: An AI assistant that reads engineering PDF documents
and answers questions about them in Arabic and English.

Files in this project:
  config.py       — all settings in one place
  ingest.py       — loads PDFs, creates vector database
  rag_pipeline.py — the RAG logic (retrieve + generate)
  app.py          — the Gradio web interface
  evaluate.py     — measures system quality
  run.sh          — shell script launcher (Linux/Git Bash)
  requirements.txt
  docs/           — put your PDFs here
  chroma_db/      — auto-created, stores vectors
  logs/           — auto-created, stores query logs


==============================================================
STEP 1 — INSTALL OLLAMA (your free local LLM)
==============================================================

1. Go to https://ollama.com
2. Download and install for Windows
3. Open Command Prompt and run:
      ollama pull mistral
   This downloads Mistral 7B (~4GB). One time only.
4. Verify it works:
      ollama run mistral "say hello"
   You should see a response. Press Ctrl+D to exit.

What is Ollama?
  Ollama runs large language models locally on your machine.
  No internet needed after download. No API costs. Ever.
  Mistral 7B is a powerful open-source model from Mistral AI.


==============================================================
STEP 2 — SET UP PYTHON ENVIRONMENT
==============================================================

Open Command Prompt in the project folder and run:

  python -m venv venv
  venv\Scripts\activate
  pip install -r requirements.txt

This installs:
  - LangChain: the framework connecting all components
  - ChromaDB: the vector database (runs locally)
  - sentence-transformers: converts text to vectors (embeddings)
  - Gradio: builds the web UI with ~10 lines of code
  - pypdf: reads PDF files
  - rich: makes terminal output look nice
  - ollama: Python client for Ollama

Installation takes ~5 minutes. The embedding model
(multilingual-e5-small, ~90MB) downloads on first run.


==============================================================
STEP 3 — GET A PDF DOCUMENT
==============================================================

You need a real technical PDF. Here are free options:

Option A (best for Siemens role):
  Search Google: "Calibre DRC user manual PDF"
  or: "EDA design rules documentation PDF"
  or: "VLSI layout guidelines PDF"

Option B (any technical PDF works):
  - Any engineering textbook PDF
  - Research paper PDF
  - Arabic technical document PDF (tests bilingual feature)

Place the PDF(s) in the docs/ folder.
Create the folder if it doesn't exist.


==============================================================
STEP 4 — RUN INGESTION
==============================================================

  python ingest.py

What happens:
  1. Reads every PDF in docs/
  2. Splits text into chunks of ~600 characters
  3. Each chunk gets converted to a vector (384 numbers)
  4. All vectors stored in chroma_db/ folder

This runs once. Re-run only when you add new PDFs.

Expected output:
  Loading PDFs...
  Reading: your_document.pdf
  Loaded 45 pages
  Total chunks created: 312
  Building vector store...
  Done! Now run: python app.py


==============================================================
STEP 5 — RUN THE APP
==============================================================

First, make sure Ollama is running:
  Open a NEW Command Prompt window and run:
    ollama serve
  Leave that window open.

Then in your project window:
  python app.py

Open browser at: http://localhost:7860

You should see the dark-themed Fasih-Docs interface.
Ask a question about your PDF. Get an answer with sources.


==============================================================
STEP 6 — RUN EVALUATION
==============================================================

  python evaluate.py

This tests your system and saves metrics to logs/eval_report.json

PUT THESE NUMBERS IN YOUR CV AND GITHUB README.
Example: "Achieved 85% response rate with <5s average latency"


==============================================================
WHAT TO LEARN WHILE BUILDING THIS
==============================================================

While you build, understand these concepts deeply.
You will be asked about them in interviews.

1. WHY DO WE CHUNK TEXT?
   LLMs have limited context windows. You cannot give a
   200-page PDF to an LLM at once. Chunking breaks it into
   pieces so you only give the RELEVANT pieces to the LLM.

2. WHAT IS AN EMBEDDING?
   A vector (list of numbers) that represents the meaning
   of a piece of text. Similar meaning = similar numbers.
   This is how "semantic search" works — not keyword matching.

3. WHAT IS A VECTOR DATABASE?
   A database optimized for storing and searching vectors.
   ChromaDB uses HNSW (Hierarchical Navigable Small World)
   algorithm for fast approximate nearest-neighbor search.

4. WHAT IS RAG?
   Retrieval Augmented Generation:
   R: Find the relevant chunks from your documents
   A: Add them to the prompt as context
   G: Generate an answer using the LLM + that context
   
   WHY RAG instead of just fine-tuning?
   - Fine-tuning bakes knowledge into model weights permanently
   - RAG keeps knowledge external and updatable
   - RAG is cheaper (no GPU training needed)
   - RAG cites sources (traceable and auditable)

5. WHAT IS MISTRAL 7B?
   An open-source LLM with 7 billion parameters from
   Mistral AI (French company). Runs on a normal laptop.
   Good at following instructions and technical questions.

6. WHAT IS LANGCHAIN LCEL?
   LangChain Expression Language. A way to chain AI
   components together using the | (pipe) operator.
   Similar to Unix pipes: command1 | command2 | command3

7. WHY SHELL SCRIPTING? (run.sh)
   Engineering teams run software from the command line.
   Shell scripts automate environment checks, startup,
   and logging. At Siemens, everything runs on Linux servers.

==============================================================
COMMON ERRORS AND FIXES
==============================================================

ERROR: "Connection refused" or "Ollama not running"
FIX: Open a new terminal and run: ollama serve

ERROR: "No module named langchain"
FIX: Make sure your venv is activated: venv\Scripts\activate

ERROR: "No PDF files found"
FIX: Put PDF files inside the docs/ folder

ERROR: Ingest is very slow
FIX: Normal. First run downloads the embedding model (~90MB)
     Subsequent runs are much faster.

ERROR: Out of memory during inference
FIX: Mistral 7B needs ~8GB RAM. Close other apps.
     Or use a smaller model: ollama pull phi3:mini
     Then change OLLAMA_MODEL = "phi3:mini" in config.py
